# PLAN DE DISEÑO: SISTEMA DE RUTEO LOGÍSTICO COMPUYA

Este documento detalla la estrategia para implementar el sistema de sugerencia de rutas desde las sucursales hasta el destino del envío.

## 1. ARQUITECTURA DE DATOS (MODELOS)

Para que el ruteo funcione, necesitamos coordenadas precisas (latitud y longitud). No basta con la dirección en texto.

*   Nueva Entidad 'Sucursal': Representa los centros logísticos desde donde salen los envíos.
    - Campos: nombre, direccion, latitud (float), longitud (float).
*   Actualización de 'Usuario' (Empresa):
    - Añadiremos latitud y longitud. Esto permitirá que, cuando una empresa se registre o actualice su perfil, usemos Leaflet en el frontend para capturar su ubicación exacta.
*   Actualización de 'Envio':
    - sucursal_id: Clave foránea a la sucursal asignada.
    - latitud_destino, longitud_destino: Capturadas del perfil del usuario destinatario al momento de crear el envío.

## 2. LÓGICA DE RUTEO (BACKEND)

Implementaremos un servicio de ruteo basado en OSRM (Open Source Routing Machine). La lógica será:

1.  Selección de Sucursal Óptima: Al crear un envío, el backend calculará la distancia lineal (o por carretera si usamos la API) entre el destino y todas las sucursales disponibles.
2.  Sugerencia: El sistema sugerirá la sucursal más cercana automáticamente.
3.  Cálculo de Ruta: El backend consultará a OSRM para obtener la "geometría" de la ruta (una lista de puntos que forman el camino por las calles).

## 3. VISUALIZACIÓN (FRONTEND CON LEAFLET)

*   Mapa del Repartidor: Usaremos Leaflet para mostrar un mapa interactivo.
*   Marcadores: Uno para la sucursal de origen (A) y otro para el destino (B).
*   Polilínea: Dibujaremos sobre el mapa el camino exacto sugerido por el backend.

---

### PASOS PROPUESTOS PARA LA EJECUCIÓN:

1.  Backend: Crear la tabla 'sucursales' en la DB y actualizar los modelos 'Usuario' y 'Envio'.
2.  Backend: Crear un 'ServicioRuteo' que se conecte con OSRM para calcular distancias y rutas.
3.  Backend: Actualizar la creación de envíos para que asigne automáticamente la sucursal más cercana.
4.  Frontend: Integrar Leaflet para mostrar el mapa en el detalle del envío del repartidor.
